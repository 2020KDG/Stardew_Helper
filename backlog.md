# 스타듀 밸리 오버레이 향후 개발 백로그 (Future Backlog)

이 문서는 추후 업데이트나 아키텍처 변경 시 고려할 수 있는 아이디어와 기능을 기록해 두는 공간입니다.

## 1. SMAPI 연동 기반의 "완전 실시간" 통신 모듈 도입 검토
현재 오버레이는 순정 게임에서도 작동할 수 있도록 세이브 파일 파싱 방식을 사용 중입니다. (게임 내 하루가 끝날 때 갱신됨).
하지만 유저에게 '1초 단위의 즉각적인 실시간 갱신'이 필요해질 경우, 별도의 SMAPI 모드를 제작하여 오버레이와 통신하는 방식을 추가 옵션으로 도입할 수 있습니다.

### 💡 구현 개념 (Architecture)
1. **오버레이 (Rust 백엔드)**
   - 백그라운드에서 로컬 웹소켓(WebSocket) 서버나 초소형 HTTP 서버(예: `localhost:8080`)를 열어두고 대기합니다.
2. **SMAPI 플러그인 (C# 모드)**
   - 게임 내 이벤트(예: `MoneyChanged`, `AchievementUnlocked`, `ItemAddedToInventory`)를 이벤트 리스너로 후킹합니다.
   - 이벤트가 발생하는 즉시, 오버레이가 열어둔 웹소켓 주소로 실시간 데이터(JSON)를 쏩니다.
3. **프론트엔드 (UI)**
   - 백엔드를 통해 신호를 전달받고, 즉시 화면의 재화나 도전과제 완료 상태를 갱신합니다.

### ✅ 장점
- 플레이어가 특정 조건(돈 벌기, 아이템 획득, 도전과제 달성)을 완료하는 **그 찰나의 순간**에 '뚜둔' 알림과 함께 오버레이 UI도 실시간으로 갱신됩니다.
- 세이브 파일 파싱 방식에 비해 지연 시간이 0에 가깝습니다.

### ⚠️ 단점 (고려사항)
- 사용자가 오버레이 앱 하나만 켜서 되는 것이 아니라, **반드시 SMAPI를 설치**하고 우리가 배포하는 `.dll` 모드 파일을 스타듀 밸리 `Mods` 폴더에 직접 넣어야 합니다.
- 순정(바닐라) 유저의 진입 장벽이 높아집니다.
- 게임이 대규모 패치(예: 1.6 -> 1.7)를 할 때마다 C# 모드 코드를 유지보수해야 할 부담이 늘어납니다.

### 🎯 도입 전략 제안
- 기본 베이스: 현재의 '세이브 감지 모델'을 기본 모드로 제공하여 누구나 쉽게 다운받아 쓸 수 있게 함.
- 고급 옵션 (Pro Version): 실시간 통신을 원하는 하드코어 유저들을 위해 "실시간 통신 플러그인"을 별도로 제공하고, 오버레이 앱의 설정 탭에서 'SMAPI 실시간 연동 모드 켜기'를 제공하는 투트랙(Two-Track) 방식 권장.
